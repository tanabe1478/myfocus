# AI要約受信箱

記事画面の「AI要約を生成」は、生成完了まで画面を占有せずSQLiteの要約キューへ追加する。

## 体験

- ボタンを押すと即座にキューへ入り、閲覧を続けられる
- 左サイドバーの「AI要約」に生成中件数と未確認件数を表示
- 一覧には `待機中`、`生成中`、`新着`、`確認済み`、`失敗` を表示
- 完了した要約を開くと未確認バッジが減る
- 失敗した記事は記事画面から再試行できる
- 過去の要約は確認後も履歴として残る

## 永続化と実行

`article_summary_jobs`テーブルにジョブ状態、エラー、要求日時、確認日時を保存する。ジョブは一件ずつ実行し、アプリ終了時に実行中だったものは次回起動時に再開する。

要約済み記事は履歴を維持するため、通常の既読記事保持期間による削除対象から除外する。
